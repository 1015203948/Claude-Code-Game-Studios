## Purpose & Player Need

主菜单是玩家每次启动游戏的第一触点。它需要在 5 秒内传达「这是一个太空帝国策略游戏」的氛围，并让玩家以最少 1 次点击进入游戏。

**核心需求**：
- 快速进入游戏（有存档时继续，无存档时新游戏）
- 设置访问（音量、震动等系统偏好）
- 版本/版权信息展示

---

## Player Context on Arrival

### 首次启动
玩家首次打开游戏，进入空白存档状态。主菜单显示「新游戏」作为主要入口，氛围预览游戏视觉风格。

### 回归玩家
玩家有存档存在。主菜单显示「继续游戏」作为主要入口（突出「回来继续你的帝国」的情感承诺）。

### 更新弹窗场景（预留）
游戏版本更新后首次进入，显示更新说明弹窗（P8 Modal）。

## Navigation Position

```
Session Start
└── 主菜单
    ├── [游戏标题]
    ├── [继续游戏] — 仅在有存档时显示
    ├── [新游戏]
    ├── [设置]
    └── [版本号 + 版权]

游戏退出 → 返回主菜单
```

## Entry & Exit Points

| 入口 | 触发条件 | 动画 |
|------|---------|------|
| App Launch | 应用启动 | 标题淡入 0.5s，按钮组依次淡入（间隔 0.1s） |
| 返回主菜单 | 退出游戏 | 场景淡出 0.3s |

| 出口 | 目的地 | 触发 |
|------|--------|------|
| 继续/新游戏 → 确认 → 执行 | 星图视图 | Tap 按钮 |
| 设置 | 设置页面 | Tap 按钮 |

## States & Variants

### State 1: 无存档

```
[新游戏] ← 主按钮（#2266CC）
[设置]   ← 次按钮
```

### State 2: 有存档

```
[继续游戏] ← 主按钮（#2266CC）
[新游戏]  ← 次按钮
[设置]    ← 次按钮
```

### State 3: 有存档 + 新游戏确认弹窗

```
┌──────────────────────────┐
│        警告               │
│                          │
│  开始新游戏将覆盖         │
│  当前存档。确定继续？      │
│                          │
│ [取消]        [确定]      │
└──────────────────────────┘
```

### State 4: 设置页面入口

从主菜单 Tap「设置」→ 进入设置页面（见 pause-menu.md）。

## Interaction Map

```
主菜单
├── [继续游戏] — Tap → [存档选择/直接加载] → 加载 → 星图视图
├── [新游戏]   — Tap
│                 ├── [无存档] → 确认弹窗 → [确定] → 加载星图（初始状态）
│                 └── [有存档] → 确认弹窗 → [确定] → 覆盖存档 → 加载星图
└── [设置]     — Tap → 设置页面
```

## Events Fired

| 事件 | 参数 | 触发时机 |
|------|------|---------|
| `OnMainMenuOpened` | — | 主菜单场景激活 |
| `OnContinueGameClicked` | — | Tap 继续游戏 |
| `OnNewGameClicked` | — | Tap 新游戏 |
| `OnNewGameConfirmed` | — | 确认弹窗确认 |
| `OnNewGameCancelled` | — | 确认弹窗取消 |
| `OnSettingsClicked` | — | Tap 设置 |
| `OnGameStarted` | `bool isNewGame` | 加载游戏时（区分新游戏/继续） |

## Transitions & Animations

| 过渡 | 时长 | 缓动 |
|------|------|------|
| 标题淡入 | 500ms | ease-out |
| 按钮组依次淡入 | 每项 100ms，间隔 100ms | ease-out |
| 按钮按下 | alpha: 1.0 → 0.7 | 50ms linear |
| 按钮释放 | alpha: 0.7 → 1.0 | 50ms linear |
| 确认弹窗出现 | 300ms | scale: 0.9→1.0 + fade |
| 确认弹窗消失 | 200ms | fade out |
| 场景切换（→星图） | 500ms | fade to black |

**禁止**：按钮不使用缩放动画（触屏手感不一致）。

## Accessibility

| 要求 | 实现 |
|------|------|
| A1: 热区 | 所有按钮热区 ≥48×48dp |
| A2: 色盲 | 按钮有图标+文字双重标识 |
| A3: 文字缩放 | 按钮文字使用 sp，支持系统缩放 |
| A4: 震动反馈 | Tap 按钮时触发轻震动 |
| 对比度 | 按钮文字与背景对比度 ≥4.5:1 |

## Acceptance Criteria

- [ ] 无存档时仅显示「新游戏」和「设置」按钮
- [ ] 有存档时显示「继续游戏」「新游戏」「设置」三按钮
- [ ] 「继续游戏」按钮为主色调（#2266CC），其余为次要色
- [ ] 所有按钮最小触控热区 48×48dp
- [ ] 按下时按钮 alpha 变为 0.7，抬起恢复
- [ ] 有存档时「新建游戏」弹出确认对话框
- [ ] 确认对话框不可点击外部关闭，必须使用按钮
- [ ] 场景切换使用 500ms 淡黑过渡
- [ ] 版本号显示在底部（小字，不干扰主操作）
- [ ] 支持 Android 系统字体缩放
