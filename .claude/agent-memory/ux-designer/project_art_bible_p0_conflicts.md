以下 5 项在 2026-04-12 的 UX 对齐审查中被标记为 P0，必须在任何 UI 实现工作开始前书面确认：

1. **星图信息三级优先级框架** — 需锁定 P1/P2/P3 分类及对应亮度阈值（P1 建议 ≥85% HSL）
2. **手机屏幕安全区地图** — 顶部 15% 状态栏、中部 60% 禁止遮挡、底部 25% 操作栏，需书面确认
3. **FPV vs TPV 双套 HUD 方案** — 两套视角信息展示方式根本不同，不可共用同一布局；FPV 驾驶舱面板风格 vs TPV 世界空间 UI
4. **虚拟摇杆安全区先于 HUD 布局锁定** — 左摇杆 80×80dp 区域及右侧操作区需先定义，HUD 元素不得进入；摇杆位置可由玩家调整，HUD 系统需响应
5. **"亮度=优先级"系统的辅助识别维度** — 10+ 单位同屏时纯亮度系统失效，需补充形状差异（菱形/圆形/方形）或动态脉动（1Hz）作为辅助维度；无障碍模式下必须有非亮度替代方案

**Why:** 这些项如未在实现前确认，会导致 Art Director 与 UI Programmer 的解读分歧，或在 QA 阶段引发系统性返工。

**How to apply:** 若用户提到要开始实现星图或驾驶舱 HUD，主动提醒这 5 项 P0 是否已确认。
